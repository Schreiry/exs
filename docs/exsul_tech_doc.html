1. მიმოხილვა

Exsul არის local-first დესკტოპ AI ასისტენტი მცირე და საშუალო ბიზნესისთვის. მისი მთავარი ეკრანი ერთიანი სამუშაო კონტექსტია: მომხმარებელი ეკრანის ნებისმიერ მდგომარეობაში იწყებს წერას, სისტემა ეძებს პროდუქტებს, აანალიზებს ბიზნეს-მონაცემებს და პასუხთან ერთად უშუალოდ სცენაზე აჩენს ბარათებსა და სურათებს. მონაცემები რჩება მოწყობილობაზე; AI პროვაიდერს ეგზავნება მხოლოდ მოთხოვნისთვის შერჩეული, ზომით შეზღუდული კონტექსტი. გაფართოებადი Action Island (Alt+C ან კურსორის სწრაფი მოძრაობა) AI, ბიზნესისა და ანალიტიკის მოქმედებებს ერთ ტიპიზებულ რეესტრში აერთიანებს.

2. არქიტექტურა

კლავიატურა / Action Island / ფაილის არჩევა → Svelte Scene → TypeScript + Zod → Tauri IPC → Rust სერვისები → SQLite + FTS5 / შეზღუდული ფაილური კონტექსტი → AI Gateway → ერთიანი შედეგების სცენა

პრინციპი: UI არ ეხება SQLite-ს პირდაპირ; პროვაიდერის გასაღები არ ტოვებს Rust backend-ს; AI-ის პასუხი ყოველთვის არასანდო მონაცემად ითვლება.

3. ტექნოლოგიური სტეკი

არქიტექტურა გამიჯნულია სცენის, ბრძანებების, მონაცემებისა და პროვაიდერების ფენებად. არჩევანის საფუძველია სწრაფი გაშვება, ოფლაინ-მუშაობა, მკაცრი ტიპიზაცია, აღდგენა და პროვაიდერების ჩანაცვლებადობა.

ფენა ტექნოლოგია რატომ ეს არჩევანი
Desktop ბირთვი Tauri 2, Rust 2021, Tokio ნატიური ფანჯარა, უსაფრთხო async I/O და მცირე რესურსული კვალი
ინტერფეისი Svelte 5, SvelteKit, TypeScript, Vite რეაქტიული void-სცენა, სწრაფი განახლება და კომპაქტური frontend
მოქმედებები Action Registry + Action Island AI/Business/Analytics მოდულები ღილაკებს მთავარი კომპონენტის შეცვლის გარეშე ამატებენ
მონაცემები SQLite, rusqlite, WAL, event ledger ლოკალური ტრანზაქციები, პროექციები, მიგრაციები, აღდგენა და სარეზერვო ასლები

კონტრაქტი Serde, Zod, cargo test/clippy, svelte-check runtime-ვალიდაცია, frontend/backend სქემების თანხვედრა და მკაცრი CI-შემოწმება

4. დანართი: სისტემის მუშაობის დეტალები

ქვემოთ ჩამოთვლილია სისტემის ძირითადი გარანტიები და გაფართოების წესები.

  • ინტერაქცია: პირველი დასაბეჭდი სიმბოლო ავტომატურად ააქტიურებს ცენტრალურ ველს; Enter აგზავნის მოთხოვნას, დასრულებული პასუხი კი ზედა, scroll-ით მართვად ისტორიაში გადადის. ჩვეულებრივი chat bubble-ები და გვერდითი ნავიგაცია არ გამოიყენება.
  • მონაცემები და ძიება: FTS5 ლოკალურად აბრუნებს მაქსიმუმ 40 შედეგს; AI-ს მხოლოდ top-8 პროდუქტის მინიმალური კონტექსტი გადაეცემა. მიგრაცია 1001 legacy-სქემას მონაცემების დაკარგვის გარეშე აახლებს; დახურვისას იქმნება სარეზერვო ასლი.
  • Action Island: იხსნება Alt+C-ით ან კურსორის shake-ჟესტით; აქვს scroll, focus trap, კლავიატურული მართვა და reduced-motion რეჟიმი. ახალი მოდული საკუთარ მოქმედებას namespaced ID-ით registerAction-ში არეგისტრირებს.
  • ფაილური კონტექსტი: native picker იღებს მაქსიმუმ 10 ტექსტურ/სტრუქტურირებულ ფაილს, თითოეულს — 2 MiB-მდე. AI-მოთხოვნაში ჯამური კონტექსტი 96K სიმბოლოთი იზღუდება და prompt-injection-ისგან დაცვისთვის არასანდო მონაცემად ინიშნება. PDF/DOCX ამ ეტაპზე არ იკითხება.
  • უსაფრთხოება და ხარისხი: გასაღებები ინახება OS keyring-ში ან environment-ში; provider API მხოლოდ Rust-იდან იძახება. AI-ს არ აქვს სრული ბაზის, SQL-ის ან shell-ის წვდომა. მძიმე ოპერაციები async-ია, ქართული ტექსტი კი მნიშვნელობაზე დაფუძნებულ დამატებით გადამოწმებას გადის.
